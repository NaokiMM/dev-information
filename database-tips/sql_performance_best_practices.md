# SQL クエリ速度改善まとめ

## 1. WHERE句の最適化
- WHERE句で関数を使わない  
  → Where部分で関数を使うとインデックスが効かなくなる。

- BETWEEN句を適切に使用する  
  → 範囲検索を効率化できる
  👉 「計算しないように書き換える」ことが本質
  👉 BETWEENはその“書き換え方法の1つ”

---

## 2. JOINの最適化

### 基本ポイント
- JOINする前に条件でデータを絞る  
- インデックスがあるカラム同士で結合する
👉 先にデータを減らしてからJOIN

### 意識すべきこと
- 不要なJOINは避ける  
- 結合順序を意識する（データ量が少ないものから）

---

## 3. 並び替え・検索の最適化
- ORDER BYは必要最小限にする  
  → ソートはコストが高い

- EXISTS句を活用する  
  → 不要なデータ取得を防ぐ
  👉 EXISTSはWHEREの中で使うのが普通

---

## 4. インデックス戦略
- 適切なカラムにインデックスを設定する  
- WHERE / JOIN / ORDER BYで使うカラムを優先する  

---

## 5. テーブル設計
- 正規化・非正規化のバランスを見直す
- アクセスパターンに合わせて設計する

正規化
👉 メリット
データがきれい・更新しやすい
👉 デメリット
JOINが増える → 遅くなる

非正規化
👉 メリット
JOIN不要 → 速い
👉 デメリット
データ重複
更新が面倒

---

## 6. INSERT最適化
- バルクインサートを使用する  
  → 1件ずつINSERTするより高速

👉 INSERT文を何回も実行するより
👉 1つのINSERTでまとめて入れる方がいい

❌ NG（遅い）
INSERT INTO users (name) VALUES ('A');
INSERT INTO users (name) VALUES ('B');
INSERT INTO users (name) VALUES ('C');

✅ OK（速い）
INSERT INTO users (name) VALUES 
('A'),
('B'),
('C');

## 7. 分析関数の活用（Window関数）
分析関数を使って処理をシンプルにする
→ JOINやサブクエリを減らせる

👉 無駄なスキャンを減らせるため高速化につながる

---

## まとめ
- 「インデックスを効かせる設計」が最重要
- 不要なデータを読まないことがパフォーマンス改善の本質
